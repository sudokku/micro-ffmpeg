### TopBar (MODIFY in Phase 2)

Add "Import" button to the right side of the TopBar:
- Button: `text-sm font-medium px-3 py-2 rounded bg-zinc-800 hover:bg-zinc-700 text-white transition-colors`
- Label: "Import" (verb only — no noun; context is obvious)
- Triggers: native `<input type="file" accept="video/*,audio/*" multiple>` via ref click

### ToolSidebar (NEW)

Narrow 40px vertical strip on the far left:
- Background: `bg-zinc-900 border-r border-zinc-800`
- Two icon buttons stacked vertically, centered, with `gap-1` between them
- Each button: 40x40px (`w-10 h-10`) — fills full sidebar width
- Active state: `bg-zinc-800 text-white ring-1 ring-blue-500` (1px blue ring inset)
- Inactive state: `text-zinc-400 hover:text-white hover:bg-zinc-800/60 transition-colors`
- Select tool icon: `MousePointer2` from lucide-react, 16px
- Blade tool icon: `Scissors` from lucide-react, 16px
- Keyboard shortcut badge: small `text-[10px] text-zinc-500` label below each icon — "V" and "B"
- No tooltip required in Phase 2 (labels visible)

### DropOverlay (NEW)

Full-viewport translucent overlay shown on drag-over:
- Position: `fixed inset-0 z-50`
- Background: `bg-zinc-950/80 backdrop-blur-sm`
- Animation in: `transition-opacity duration-150 ease-in` — opacity 0 → 1
- Animation out: `transition-opacity duration-150 ease-out` — opacity 1 → 0
- Center content: flex column, centered, gap-4
- Upload icon: `Upload` from lucide-react, 48px, `text-zinc-400`
- Heading: "Drop video or audio files here" — 20px / weight 600 / `text-white`
- Border: `border-2 border-dashed border-zinc-600 rounded-xl` as an inset indicator (margin 24px from all edges)

### EmptyState (NEW)

Shown in the middle region when `clips` is empty:
- Position: centered in `flex-1` main region (`flex items-center justify-center`)
- Icon: `Film` from lucide-react, 40px, `text-zinc-700`
- Heading: "No clips yet" — 20px / weight 600 / `text-zinc-300`
- Body: "Drop files anywhere or click Import to get started" — 14px / weight 400 / `text-zinc-500`
- Layout: flex column, `items-center gap-3`
- No button in empty state — Import button is already in TopBar

### ClipAction (NEW)

Custom renderer passed to `getActionRender` — renders inside each clip's bounding box in the timeline:

**Structure (outermost to innermost):**
```
div (full bounds, overflow-hidden, rounded-[2px], cursor depends on active tool)
  ├── color background: bg = clip.color at 85% opacity (no overlay — full saturation looks best on dark timeline bg)
  ├── selection ring: conditional — `outline outline-2 outline-offset-[-2px] outline-white` when selected
  ├── thumbnail layer: `flex flex-row h-full` — img elements tiled horizontally, each `h-full object-cover flex-1`
  │     OR shimmer layer when thumbnailUrls is empty:
  │     `animate-pulse h-full w-full` with bg = clip.color + '40' (25% opacity shimmer)
  └── label overlay: `absolute inset-0 flex items-center px-1` (bottom half only via gradient)
        gradient: `absolute inset-0 bg-gradient-to-t from-black/60 to-transparent`
        text: `relative truncate text-xs font-semibold text-white/90 leading-none`
        format: "{filename}  {M:SS}" — two spaces between; duration dropped if clip width < 96px
```

**Cursor behavior:**
- Select tool active: `cursor-pointer` (default)
- Blade tool active: `cursor-crosshair`

**Thumbnail tiling:**
- Each `<img>` in the array covers equal width: `flex-1 h-full object-cover object-center`
- Tiling fills the entire clip width seamlessly
- No gaps between tiles

### Track Row Labels (NEW — cosmetic overlay)

The `@xzdarcy/react-timeline-editor` does not render row labels automatically. A cosmetic overlay renders "Video" / "Audio" labels:
- Position: absolutely positioned overlay over the timeline's left rule column
- Each label: 14px / weight 600 / `text-zinc-500`
- Alignment: vertically centered within each track row
- Implementation: fixed height rows matching the library's default row height (40px per row)

## Interaction Contract

### File Import — Drag and Drop (IMPT-01)

| State | Visual |
|-------|--------|
| No drag | Normal app layout — no overlay |
| File dragging over window | `DropOverlay` fades in (150ms) over entire viewport |
| Files dropped | `DropOverlay` fades out (150ms); clips appear on correct tracks |
| Non-file drag (text, link) | Overlay does NOT appear — guard with `e.dataTransfer?.types.includes('Files')` |
| Unsupported file type | Silently ignored — no error message in Phase 2 |

### File Import — File Picker (IMPT-02)

| State | Visual |
|-------|--------|
| Idle | "Import" button in TopBar, `bg-zinc-800` |
| Hover | `bg-zinc-700` transition |
| Active (picker open) | No visual change — native OS dialog takes focus |
| File selected | Clip appears on timeline; button returns to idle |

### Tool Selection (ToolSidebar)

| Event | Result |
|-------|--------|
| Click Select button | `ui.activeTool = 'select'`; Select button gets active ring; cursor on timeline clips = `cursor-pointer` |
| Click Blade button | `ui.activeTool = 'blade'`; Blade button gets active ring; cursor on timeline clips = `cursor-crosshair` |
| Press V | Select tool activated (same visual result as click) |
| Press B | Blade tool activated (same visual result as click) |

### Timeline Clip Operations

| Operation | Trigger | Visual Feedback |
|-----------|---------|----------------|
| Select clip | Click clip (Select mode) | `outline-white outline-2` ring appears; previous selection ring removed |
| Deselect clip | Click empty timeline area | Ring removed |
| Trim clip | Drag left/right edge (library handles drag UI) | Clip width changes live; `onActionResizeEnd` commits to store |
| Move/reorder | Drag clip body (library handles drag UI) | Clip moves live; `onActionMoveEnd` commits to store |
| Split clip | Click clip (Blade mode) | Clip splits into two adjacent clips at click point; both get same color as original; `cursor-crosshair` maintained |
| Delete clip | Select → press Delete or Backspace | Clip removed immediately; no animation; no confirmation |

### Undo/Redo

| Shortcut | Action | Visual |
|----------|--------|--------|
| Cmd+Z | Undo last clip operation | Clip state reverts; no toast or notification in Phase 2 |
| Cmd+Shift+Z | Redo | Clip state restored; no notification |
| No more history | No-op — `undo()`/`redo()` silently does nothing | No error state |

### Thumbnail Loading States

| State | Visual |
|-------|--------|
| Extracting (thumbnailUrls is empty) | Shimmer: `animate-pulse` with clip color at 25% opacity fills full clip width |
| Loaded (thumbnailUrls has entries) | Thumbnail images tile to fill clip width; shimmer replaced in-place |
| No layout shift | Clip occupies full width from the moment it is added to the store |

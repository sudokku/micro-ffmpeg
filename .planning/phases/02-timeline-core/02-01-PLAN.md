- Add `StoreActions` interface listing all action signatures:

```typescript
export interface StoreActions {
  addClip: (file: File, trackId: 'video' | 'audio', duration: number) => void
  moveClip: (clipId: string, newStart: number, newEnd: number) => void
  trimClip: (clipId: string, newStart: number, newEnd: number) => void
  splitClip: (clipId: string, splitTime: number) => void
  deleteClip: (clipId: string) => void
  selectClip: (clipId: string | null) => void
  setActiveTool: (tool: 'select' | 'blade') => void
}
```

Update `StoreState` to extend or include `StoreActions` (either `StoreState & StoreActions` or add actions directly to StoreState).

**Step 2: Implement actions in `src/store/index.ts`**:

Import `CLIP_COLORS` from `../constants/clipColors`. Maintain a module-level `let colorIndex = 0` counter.

Implement each action inside the `create()` call using `set()`:

- `addClip(file, trackId, duration)`: create clip with `crypto.randomUUID()` id, compute startTime from last clip on track (or 0), assign `color: CLIP_COLORS[colorIndex++ % CLIP_COLORS.length]`, set `thumbnailUrls: []`. Add to `clips` record and append id to `tracks[trackId].clipIds`.

- `moveClip(clipId, newStart, newEnd)`: update `clips[clipId].startTime` and `clips[clipId].endTime`. Immutable update: spread existing clip.

- `trimClip(clipId, newStart, newEnd)`: same as moveClip — update startTime and endTime.

- `splitClip(clipId, splitTime)`: guard — if `splitTime <= clip.startTime + 0.01 || splitTime >= clip.endTime - 0.01` return unchanged state. Create two new clips (left and right) with `crypto.randomUUID()` ids. Left: `{ ...original, id: leftId, endTime: splitTime }`. Right: `{ ...original, id: rightId, startTime: splitTime }`. Remove original from clips record (destructure out). Replace original id in track.clipIds with [leftId, rightId] at same index.

- `deleteClip(clipId)`: remove from `clips` record (destructure out). Filter from `tracks[clip.trackId].clipIds`.

- `selectClip(clipId)`: `set({ ui: { ...state.ui, selectedClipId: clipId } })` — this modifies the ui slice which is excluded from Zundo history.

- `setActiveTool(tool)`: `set({ ui: { ...state.ui, activeTool: tool } })` — same exclusion.

**Critical patterns:**
- All clip mutations (addClip, moveClip, trimClip, splitClip, deleteClip) MUST use the `set(state => ...)` form so Zundo captures them.
- selectClip and setActiveTool mutate only `ui` which is excluded from Zundo partialize — they do NOT appear in undo history.
- The color index counter is module-level (outside the store) so it persists across undo/redo cycles.

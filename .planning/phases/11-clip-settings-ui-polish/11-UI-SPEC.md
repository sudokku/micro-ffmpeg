## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. `px-1.5 py-0.5` badge, `p-1` icon buttons) |
| sm | 8px | Compact element spacing (e.g. `gap-2`, `py-1.5`) |
| md | 16px | Default section padding (e.g. `px-4` TopBar, `px-3` settings panel sections) |
| lg | 24px | Section breathing room (e.g. `pt-3` section header top) |
| xl | 32px | Layout gaps (not currently used at component level) |
| 2xl | 48px | Major section breaks (not used; reserved for future layout) |
| 3xl | 64px | Page-level spacing (not applicable — app is full-bleed) |

Exceptions:
- Sidebar settings panel uses `px-3` (12px) for internal row padding — deliberate narrower gutter to fit 280px panel width
- TopBar height is fixed at `h-12` (48px) — matches minimum touch target height
- Timeline height changes from `37vh` to `28vh` (source: CONTEXT.md decision)

**Source:** Codebase scan of `ClipSettingsPanel.tsx`, `TopBar.tsx`, `AppShell.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / slider labels | 12px (text-xs) | 400 (normal) | 1.5 |
| UI labels / buttons | 14px (text-sm) | 400 (normal) | 1.4 |
| Section headings | 12px (text-xs) | 600 (semibold) | 1.2 |
| Display / empty state | 20px (text-xl) | 600 (semibold) | 1.2 |

Notes:
- Exactly 3 sizes in active use: 12px, 14px, 20px. A 4th (section header) reuses 12px at semibold — differentiated by weight and `uppercase tracking-wide`, not size.
- Weights: 400 (normal) for body copy, data labels, and button text; 600 (semibold) for section headers and empty state heading.
- Section header pattern established in existing code: `text-xs font-semibold text-zinc-400 uppercase tracking-wide` — all new section headers (PLAYBACK, TRANSFORM) must use this exact class string.

**Source:** Codebase scan of `ClipSettingsPanel.tsx`, `TopBar.tsx`, `EmptyState.tsx`.

## Color

Accent reserved for:

Inactive button state: `bg-zinc-800 text-zinc-300 hover:bg-zinc-700`
Active button state: `bg-blue-600 text-white`

## Component Inventory

### New: Segmented Button Row (inline, no new file)

Used for Speed and Rotation controls. Built inline in `ClipSettingsPanel.tsx`.

```
[ 0.25× ][ 0.5× ][ 1× ][ 2× ][ 4× ]
```

Interaction contract:
- Render as a `<div className="flex rounded overflow-hidden border border-zinc-700">` containing `<button>` elements
- Each button: `flex-1 text-xs py-1.5 font-normal transition-colors`
- Inactive: `bg-zinc-800 text-zinc-300 hover:bg-zinc-700`
- Active (matches store value): `bg-blue-600 text-white`
- Immediate commit on click (no local state needed — discrete values)
- Bulk-apply via `selectedClipIds.length > 1` fan-out (same pattern as existing commit handlers)

### New: Flip Toggle Buttons (inline, no new file)

Two side-by-side icon+label toggle buttons for Flip H and Flip V.

```
[ ⇔ H ][ ⇕ V ]
```

Interaction contract:
- Render as `<div className="flex gap-2">` with two `<button>` elements
- Each button: `flex items-center gap-1 px-2 py-1.5 rounded text-xs font-normal transition-colors`
- Inactive: `bg-zinc-800 text-zinc-300 hover:bg-zinc-700`
- Active: `bg-blue-600 text-white`
- Use Lucide `FlipHorizontal` and `FlipVertical` icons (size 14)
- Immediate commit on click

### Modified: Volume Slider

- Commit-on-release pattern (same as Blur/Brightness — `onPointerUp` + `onTouchEnd`)
- Range: 0–200, step 5
- Display: `"100%"` format — append `%` to integer value
- `accentColor: '#3b82f6'`

### Modified: Hue Slider

- Commit-on-release pattern (same as existing sliders)
- Range: -180 to +180, step 1
- Display: `"0°"` format — append `°` to integer value
- `accentColor: '#3b82f6'`

### Modified: ClipSettingsPanel container

- Width: `w-70` (280px) — was `w-60` (240px) per CONTEXT.md decision
- Audio clip guard: when `clip.trackId === 'audio'`, render only the PLAYBACK section (Speed + Volume); hide TRANSFORM, FILTERS, CROP, RESIZE
- Detection: `clip.trackId === 'audio'` (not MIME type)

### Modified: AppShell timeline height

- Timeline div: `style={{ height: '28vh' }}` — was `'37vh'`

## Panel Section Order

Full section order in `ClipSettingsPanel` (top to bottom):

1. **Header** — clip name + multi-select badge (existing, unchanged)
2. **PLAYBACK** — Speed segmented row (new), Volume slider (new)
3. **TRANSFORM** — Rotation segmented row (new), Flip H/V buttons (new), Hue slider (new)
4. **FILTERS** — Blur, Brightness, Contrast, Saturation sliders (existing, unchanged)
5. **CROP** — X/Y/W/H numeric inputs (existing, unchanged)
6. **RESIZE** — Width/Height with aspect-lock (existing, unchanged)

Audio clips show only Header + PLAYBACK. All other sections hidden via `clip.trackId === 'audio'` guard.

## Interaction States

| Control | Default State | Active/Engaged State | Error State |
|---------|--------------|---------------------|-------------|
| Segmented button (speed/rotation) | `bg-zinc-800 text-zinc-300` | `bg-blue-600 text-white` | N/A (always valid) |
| Flip toggle button | `bg-zinc-800 text-zinc-300` | `bg-blue-600 text-white` | N/A |
| Range slider | native browser track | `accentColor: '#3b82f6'` thumb | N/A |
| Numeric input | `bg-zinc-800 border-zinc-700 text-white` | focused (browser default outline) | N/A — invalid parsed silently (matches existing pattern) |

Hover states: all buttons use `hover:bg-zinc-700` on inactive state; active buttons have no hover variant (already at accent — no change).

Commit timing:
- Segmented buttons, flip toggles: immediate on click
- Sliders (volume, hue): commit on `onPointerUp` / `onTouchEnd`; display local state during drag
